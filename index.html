Rob Muscat: how I build websites with OpenClaw on WhatsApp

I message my assistant (OpenClaw) in WhatsApp like a normal chat. OpenClaw runs on a server and can: write code files, run commands, create GitHub repos, and deploy to services like Render.

What it feels like
Texting instructions → a website appears.
What’s actually happening
A bot on a server edits files + pushes to GitHub + triggers deploy.

The workflow (high level)

  1. Describe the site “Make a tribute site” / “Make a financial calculator” / “Make a landing page”
  2. OpenClaw generates the files HTML/CSS/JS (or a framework if needed), plus assets.
  3. Commit + push to GitHub The repo becomes the source of truth. Every change is tracked.
  4. Deploy Render auto-deploys on push, or GitHub Pages hosts static sites.
  5. Iterate via chat “Make it look better on mobile.” “Replace placeholders with real quotes.”

Architecture (simple)

WhatsApp message
   ↓
OpenClaw (server)
   ↓
Code in a project folder
   ↓
GitHub repo
   ↓
Render/GitHub Pages deploy

A real example (short)

What I text

Make a website dedicated to Jackie Mason.
Put it on GitHub.
Deploy it to Render.
Now make it look better on mobile.

What OpenClaw does

  • Creates a project folder
  • Writes HTML/CSS/JS
  • Commits + pushes to GitHub
  • Creates a Render Blueprint file (render.yaml)
  • Updates styling until it looks right

Deploy options

Render (recommended for quick deploys)

  • Connect GitHub repo once
  • Enable auto-deploy
  • Every push redeploys automatically

Hostinger / shared hosting (great for static sites)

  • Upload the files (or sync via Git)
  • Cheapest + simplest for pure HTML/CSS/JS

Safety + good practices

  • Don’t put secrets in chat or code (API keys, passwords). Use environment variables.
  • Use GitHub so every change is tracked and reversible.
  • Start simple (static site), then add a backend only if needed.

This page is intentionally high-level: it explains the concept without sharing any sensitive configuration.
